### AWS Data Engineering Document Summary

This document introduces key AWS services for real-time data streaming, database migration, and various data ingestion scenarios, explaining their core functionalities and use cases.

---

### 1. Key Concepts

*   **Real-time Message Streaming:** The fundamental idea of applications sending and receiving messages instantly, exemplified by Apache Kafka and its managed AWS counterpart, Amazon MSK.
*   **Kafka Core Components:**
    *   **Topic:** A category or feed name for messages.
    *   **Partition:** A way to split a topic to distribute load, enabling faster processing, scalability, and parallel reading.
    *   **Broker:** A Kafka server that stores data, providing high availability and fault tolerance.
*   **Managed vs. Serverless:** Different operational models for cloud services, offering trade-offs between control/predictability (Managed/Provisioned) and automated scaling/reduced overhead (Serverless).
*   **Database Migration Strategies:**
    *   **Full Load:** Copying all existing data once.
    *   **Change Data Capture (CDC):** Continuously tracking and replicating ongoing data changes (inserts, updates, deletes).
    *   **Schema Conversion:** The process of translating database schemas (tables, views, procedures) between different database engines.
*   **Large-scale Data Transfer:** Solutions for moving vast amounts of data efficiently, whether over the network or physically.
*   **Managed File Transfer:** Providing secure and managed file transfer protocols (SFTP/FTPS/FTP) as a service.

---

### 2. Important AWS Services

*   **Amazon MSK (Managed Streaming for Kafka):** A fully managed service for Apache Kafka clusters.
    *   **MSK Provisioned:** User defines brokers, instance types, storage. Best for predictable workloads.
    *   **MSK Serverless:** AWS automatically manages capacity and scaling. Best for variable workloads.
    *   **MSK Connect:** A feature to integrate MSK with other systems (e.g., S3, Redshift, ElasticSearch, MySQL) using pre-built connectors without custom code.
*   **Amazon Kinesis:** AWS's native streaming data service, offering simpler setup and tight AWS integration.
*   **AWS DMS (Database Migration Service):** Migrates databases with minimal downtime, supporting both full load and continuous CDC. Supports various source and target endpoints (e.g., Oracle to RDS PostgreSQL, MySQL to S3).
*   **AWS Schema Conversion Tool (SCT):** Converts heterogeneous database schemas (e.g., Oracle syntax to PostgreSQL syntax) before migration.
*   **AWS DataSync:** Accelerates and simplifies moving large amounts of files between on-premises storage and AWS storage services (like S3, EFS, FSx).
*   **AWS Transfer Family:** Provides fully managed support for SFTP, FTPS, and FTP protocols, integrating directly with Amazon S3 for storage.
*   **AWS Snow Family:** Physical devices for transferring petabytes of data to and from AWS when internet connectivity is limited or too slow.
    *   **AWS Snowcone:** Small, portable for remote edge locations.
    *   **AWS Snowball Edge:** Medium-sized devices for 50-100 TB data migration and edge computing.
    *   **AWS Snowmobile:** A truck-sized container for exabyte-scale data transfers.
